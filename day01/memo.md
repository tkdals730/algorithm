# 수업 관련 메모
- 큐 : 인큐(자료넣기) 디큐(자료 뺴기)
- 스택 : 푸시(스택에 자료넣기) , 팝(스택에 자료뺴기)

| 용어 | 쉬운 설명 |
|------|-----------|
| **알고리즘 (Algorithm)** | 문제를 푸는 순서를 적어둔 레시피. "재료 → 볶기 → 간 맞추기"처럼 단계별 절차 |
| **자료구조 (Data Structure)** | 데이터를 효율적으로 담는 그릇의 종류. 스택·큐·그래프 등이 각각 다른 모양의 그릇 |
| **스택 (Stack)** | 접시를 쌓는 것처럼 마지막에 올린 것을 먼저 꺼내는 구조 (LIFO) |
| **큐 (Queue)** | 버스 정류장 줄처럼 먼저 온 사람이 먼저 타는 구조 (FIFO) |
| **그래프 (Graph)** | 지하철 노선도처럼 **점(노드)**과 **선(간선)**으로 연결 관계를 표현한 것 |
| **노드 (Node)** | 그래프에서 각각의 점. 지하철역, 도시, 로봇 위치 등이 모두 노드가 될 수 있음 |
| **간선 (Edge)** | 노드와 노드를 잇는 선. 지하철 노선, 도로 등 |
| **BFS** | 물결이 퍼지듯 가까운 곳부터 차례로 탐색하는 방법 (너비 우선 탐색) |
| **DFS** | 미로에서 한 길을 끝까지 가다가 막히면 되돌아오는 방법 (깊이 우선 탐색) |
| **재귀 (Recursion)** | 함수가 자기 자신을 다시 호출하는 것. "거울 앞에 거울을 놓으면 끝없이 반복되는 것"과 비슷 |
| **방문(visited)** | 탐색 중 이미 지나간 노드를 기록해 두는 것. 미로에서 왔던 길에 표시를 남기는 것 |
| **LIFO / FIFO** | Last In First Out(나중에 넣은 것 먼저) / First In First Out(먼저 넣은 것 먼저) |
